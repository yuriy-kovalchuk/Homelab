# K3s Installation Script

This script automates the installation of a K3s cluster with optional Cilium networking. It supports different installation modes, including:

- **Single Master Node**
- **Single Master with Worker Nodes**
- **High Availability (HA) Cluster** (Not fully supported yet)

## Prerequisites

Before running this script, ensure that:

- All nodes have SSH access configured with the specified user.
- Swap is disabled on all nodes.
- The necessary dependencies, including `gum` for interactive prompts, are installed.
- The required ports are open for communication between nodes.

## Variables

The script defines several environment variables that should be adjusted as needed:

```bash
K3S_VERSION="v1.31.6+k3s1"  # Adjust version as needed
CILIUM_VERSION="1.16.7"     # Adjust version as needed

MAIN_MASTER_NODE="192.168.0.114"
MASTER_USER="master"
MASTER_NODES=("192.168.0.134")  # List of additional master nodes
WORKER_NODES=("192.168.0.133")  # List of worker nodes
K3S_CLUSTER_TOKEN="YOUR_CLUSTER_TOKEN"  # Replace with your desired K3s cluster token
```

## Installation Steps

1. **Run the Script**

   ```bash
   chmod +x install_k3s.sh
   ./install_k3s.sh
   ```

2. **Select the installation mode** when prompted:
   - `SINGLE_MASTER`: Installs K3s on a single master node.
   - `SINGLE_MASTER_WITH_WORKERS`: Installs K3s on a master node and adds worker nodes.
   - `HA_CLUSTER`: Attempts to set up a high-availability cluster (currently not supported).

3. **Confirm the configuration** when prompted.

4. **The script performs the following tasks:**
   - Disables swap on all nodes.
   - Installs K3s on the master node.
   - Installs K3s on worker nodes (if applicable).
   - Installs Cilium CLI on master nodes.
   - Installs Cilium CNI for networking.
   - Joins additional master nodes (if applicable).

## Functions

The script includes several functions for modular installation:

- `disable_swap`: Disables swap on all nodes.
- `install_k3s_single_master`: Installs K3s on the primary master node.
- `install_k3s_worker`: Installs K3s on worker nodes.
- `install_k3s_extra_master`: Adds extra master nodes to a cluster.
- `install_cilium_cli`: Installs the Cilium CLI on master nodes.
- `install_cilium_cni`: Deploys Cilium as the CNI plugin.

## Notes

- Ensure that `gum` is installed before running the script:
  ```bash
  brew install gum  # macOS
  sudo apt install gum  # Ubuntu/Debian
  ```
- The script assumes SSH access to all nodes with passwordless authentication.
- The `HA_CLUSTER` mode is currently not fully supported.

## Troubleshooting

- If the script fails, check the logs generated by `gum log`.
- Verify SSH access and permissions.
- Ensure that firewall rules allow communication between nodes on required ports.
